で、緑化活動って何

緑化活動 ― GitHub で何か貢献すること

活動した日は、"Contributions calendar" が緑色に染まる！

  • リポジトリにコミットしてプッシュする。
  • バグ報告、機能改善要望などの "Issue" をたてる。
  • バグ修正、機能追加などの"Pull request" をなげる。

半年くらい継続してます

2014 年の 6 月半ばくらいから始めてみた。

緑化活動するうえで決めたルール

  • 時々、一切プログラミングしない日をつくる (月に 2 ~ 4 日程度)。
    • 同じことをやってると考え方が狭くなる気がする。
  • ゴミみたいなコードでもとりあえず上げる。

なぜ緑化しはじめたのか

「つくりたいけど続かない」からの脱出

色々つくってみたいものはあるけど、なかなか継続できない。

まずは継続することを目標に GitHub で緑化活動をしてみる。

緑化されることで、作業が可視化されてモチベーションになる。

日頃からコードを書いて蓄積しておきたい

やっつけで書いて、そのままゴミ箱行きになることが多い。

  • ゴミみたいなコードなので外に見せたくない。
  • というか手元にも残したくない。
  • でも、案外後で使う場面があったりする。

ゴミでもいいので一度公開すると、アウトプットの敷居が下がる。

汚くても改善していけばいい。

緑化活動で何が起こるか

継続してると、なんとなく動くものもできてくる

最近やってるタスク。

  • mitsuse/bombay-vim ... Vim のカラーテーマ
  • mitsuse/perceptron-go ... とても単純な学習アルゴリズムの Golang 実装
  • mitsuse/kompl ... N-gram ベースの入力補完サーバー

GitHub はSNS なので pull request だしてみたり

  • fatih/vim-go ... Golang 向け Vim プラグイン

bombay-vim

かの有名なテキストエディタ、 Vim の自作カラーテーマ。

  • 標準にあるカラーテーマ desert が好きだったが見づらいので自作。
  • 若干視認性があがった。
  • モチベーションを上げるための道具磨き大事。

perceptron-go

単純だけどそこそこの性能を出せる、学習アルゴリズムの一つ。

  • データがあれば、メールのスパムフィルターみたいな分類器をつくれる。
  • 動くけど、まだ実際に使うには未実装部分多い、テストコードない。
  • Golang で実装。

Kompl

文脈に合わせて自然な単語をサジェストする機能を提供するサーバー。

  • Vim から neocomplete 経由で叩くために実装中。
  • 直前の単語 N-gram + 入力中の単語の接頭辞から予測。
  • 複数候補サジェストされ、より自然なものほど上位にくる。
  • メモリと速度との戦い。
  • これも Golang で実装。

分かったのは、継続するのは案外簡単ということ

  • ただ緑色に染めるのは簡単。
  • でも、続ければそれなりに結果がついてくる。
